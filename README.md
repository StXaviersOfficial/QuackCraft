# 🦆 QuackCraft

An original voxel building & survival sandbox for **Android phones**. Built with **Godot 4.3**, optimized for low/mid-end devices.

> **Not affiliated with Minecraft.** All textures, sounds, mob designs, and code are 100% original voxel/pixel art — generic block names like "stone", "dirt", "torch" are used descriptively, but no Minecraft assets, textures, sounds, or specifically-iconic designs (the green exploding mob, the teleporting shadow mob, etc.) are copied or derived from.

---

## 📦 What is this?

QuackCraft is a first-person voxel sandbox where you:

- Explore a procedurally-generated world with 5 biomes (Plains, Forest, Desert, Snowy, Mushroom)
- Mine and place 100+ different blocks
- Craft tools, weapons, and armor
- Survive day/night cycles with hunger and health
- Fight original hostile mobs at night
- Build whatever you can imagine

Designed from the ground up to **run smoothly on a 3–4 year-old budget Android phone**, not just flagships.

## 🏗️ Build locally

### Requirements

- [Godot 4.3-stable](https://godotengine.org/download/) (the .NET version is NOT required — pure GDScript)
- Android export templates (install via Godot Editor → Editor → Manage Export Templates → Install from official mirror)

### Steps

1. Clone this repo: `git clone https://github.com/StXaviersOfficial/QuackCraft.git`
2. Open Godot 4.3, click **Import**, navigate to the cloned folder, select `project.godot`.
3. Click **Import** — the project will open in the editor.
4. To run on desktop: press **F5** (or click the play button).
5. To build an APK: **Project → Export → Android → Export Project**.

### Regenerating assets

The texture atlas and sound effects are generated by Python scripts (the PNG/WAV files are also committed for convenience, but you can regenerate them):

```bash
pip install Pillow
python scripts_gen/gen_atlas.py     # -> assets/textures/atlas.png
python scripts_gen/gen_sounds.py    # -> assets/sounds/*.wav
```

## 🚀 Auto-publish pipeline

Every push to `main` (or `master`) automatically triggers the [**Build Android APK**](.github/workflows/build-android.yml) GitHub Actions workflow.

What it does:

1. Checks out the repo.
2. Regenerates the texture atlas & sounds from the Python generators (and commits them back if changed).
3. Downloads Godot 4.3-stable Linux headless binary + Android export templates.
4. Configures the keystore:
   - If the repo owner has set `ANDROID_KEYSTORE_BASE64` (etc.) secrets, uses that release keystore.
   - Otherwise, falls back to a debug keystore (still produces an installable APK).
5. Runs `godot --headless --export-release "Android" build/QuackCraft.apk`.
6. Uploads the APK as a workflow artifact (90-day retention).
7. Creates/updates a GitHub Release with an auto-incremented version tag (`v0.0.1` → `v0.0.2` → ...) and attaches the APK.

**Result:** every push to `main` produces a fresh, installable APK on the [Releases page](https://github.com/StXaviersOfficial/QuackCraft/releases) with zero manual steps.

### Setting up signing secrets (optional)

To produce a properly-signed release APK (instead of a debug-signed one), the repo owner adds these in **Settings → Secrets and variables → Actions**:

| Secret name | Value |
|---|---|
| `ANDROID_KEYSTORE_BASE64` | Base64-encoded `.keystore` file (`base64 -w0 release.keystore`) |
| `ANDROID_KEYSTORE_PASSWORD` | The keystore's password |
| `ANDROID_KEY_ALIAS` | The key alias inside the keystore |
| `ANDROID_KEY_PASSWORD` | The key's password |

To generate a keystore locally:

```bash
keytool -genkey -keystore release.keystore -alias quackcraft \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -dname "CN=QuackCraft,O=StXaviers,C=US"
base64 -w0 release.keystore  # paste this as ANDROID_KEYSTORE_BASE64
```

If these secrets are absent, the build still succeeds with a debug keystore.

## 🧱 What's implemented

### ✅ Core systems

- ✅ Chunked voxel world (16×128×16 chunks)
- ✅ Multithreaded-ready chunk streaming with render-distance slider
- ✅ Face-culled greedy meshing (one draw call per chunk)
- ✅ Single 256×256 texture atlas (no texture swaps)
- ✅ Procedural terrain with FastNoiseLite (height, temperature, moisture)
- ✅ 5 biomes: Plains, Forest, Desert, Snowy, Mushroom
- ✅ 3D-noise cave generation
- ✅ Depth-distributed ore veins (coal → iron → gold → gem)
- ✅ Tree generation (6 wood types) + grass, flowers, cacti, mushrooms
- ✅ Water and lava (fluid rendering, no flow simulation in v1)
- ✅ World save/load (single save slot, persisted to `user://`)
- ✅ Day/night cycle (~20 min) with sky color gradient, sun, moon
- ✅ Fog distance control

### ✅ Player & controls

- ✅ First-person camera with pitch/yaw
- ✅ Touch joystick + drag-look + jump button
- ✅ Hotbar (9 slots) with tap-to-select and swipe scroll
- ✅ Block break with crack overlay (10 stages)
- ✅ Block place on hit-face
- ✅ Tool-tier mining speed (wood < stone < iron < gold < gem)
- ✅ Required-tier gating (wood can't mine gem ore)
- ✅ Health + hunger with regen when fed
- ✅ Fall damage, lava damage, drowning
- ✅ Death + respawn at world spawn

### ✅ Blocks (100+)

- Terrain: Grass, Dirt, Stone, Cobblestone, Sand, Gravel, Clay, Snow, Ice, Sandstone, Bedrock, Mycel
- Ores: Coal, Iron, Gold, **Gem** (top-tier), Emerald, Lapis, **Spark** (redstone-equivalent, ore only in v1)
- Wood: 6 types (Oak, Birch, Spruce, Jungle, Acacia, Dark Oak) × (Log, Planks, Leaves, Sapling)
- Liquids: Water, Lava (animated tiles)
- Building: Bricks, Stone Bricks, Glass, 14 Wool colors, 14 Terracotta colors, Bookshelf, Hay Bale, Ladder
- Functional: Crafting Table, Furnace, Chest, Door, Trapdoor, Bed, Torch, Rune Altar, Brew Stand
- Nature: Tall Grass, 6 Flowers, 2 Mushrooms, Cactus, Reed, Vines, Lily Pad, Snow Layer

### ✅ Items

- Tools: Pickaxe / Axe / Shovel / Sword / Hoe in Wood, Stone, Iron, Gold, Gem tiers
- Armor: Helmet / Chestplate / Leggings / Boots in Leather, Iron, Gold, Gem
- Combat: Bow + Arrows + Shield (defined; full animation is stretch)
- Food: Bread, Cooked Beef/Pork/Chicken, Apple, Carrot, Potato, Baked Potato, Melon, Berries
- Materials: Stick, String, Feather, Leather, Bone, Spark Powder, Goo Ball, Pearl, Flint, Coal, Iron/Gold Ingots, Gem, Emerald, Lapis Dust
- Tools: Bucket (Empty/Water/Lava), Flint & Steel, Compass, Map, Clock

### ✅ Mobs

- Passive: Cow, Pig, Chicken, Sheep, Rabbit (wander, flee when hit)
- Hostile (spawn in darkness/caves):
  - **Shambler** — slow melee chaser (original design)
  - **Bonewalker** — ranged arrow shooter (original design)
  - **Web Crawler** — wall-climbing melee (original design)
  - **Shade** — short-range teleport-blink attacker (original)
  - **Husk Bomber** — original creature, hisses & detonates (NOT the green Minecraft mob — fully original design, color, VFX, and audio)

### ✅ Audio (all original, synthesized)

- Footstep variants per surface (dirt/stone/wood/sand/snow)
- Dig / place / hurt / attack / explosion / chew / pickup / splash
- Door / chest / bow draw+release / furnace loop
- Ambient biome

### ✅ UI/HUD

- Hotbar with selection highlight
- Health bar (low-health red flash)
- Hunger bar
- Day/time clock display
- Inventory overlay (2x2 quick-craft always, 3x3 when near a crafting table)
- Mining crack overlay (10 stages)
- Settings: render-distance slider, particle-density toggle, FPS cap, sensitivity, volume

## 🛠️ Tech stack

| Component | Choice | Why |
|---|---|---|
| Engine | Godot 4.3-stable | Free, open-source, mature Android pipeline, GDScript is fast to iterate |
| Renderer | Mobile (Forward+) | Tuned for phone GPUs |
| Scripting | GDScript (statically typed) | Fast iteration, no compile step |
| World | 16×128×16 chunks | Standard size, easy to thread |
| Meshing | Face-culled, single atlas | Minimal draw calls |
| Audio | Procedural WAV | All original, tiny file size |
| CI | GitHub Actions | Free for public repos, reliable |
